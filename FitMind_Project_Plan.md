# FitMind 智能健身教练平台 - 完整落地规划书

## 一、 项目愿景与目标
FitMind 旨在打造一个以 AI 智能体为核心驱动力的动态个性化健身 Web 平台。通过接入先进的 AI 模型，结合用户身体数据和实时反馈，提供动态调整的训练与饮食方案，并辅以数据可视化、社区和成就系统，解决普通用户“不知道怎么练、不知道怎么吃、难以坚持”的痛点。

## 二、 核心功能模块划分与里程碑阶段

项目将分为三个主要阶段进行敏捷开发，确保核心功能优先落地，逐步完善用户体验。

### 第一阶段：MVP (最小可行性产品) - 核心基础搭建
**目标：** 实现用户基础功能，跑通 AI 核心流程（生成静态或初步计划）。

1.  **基础设施与环境搭建**
    *   数据库设计与建表 (MySQL, Redis)
    *   Spring Boot 3 后端脚手架搭建与配置
    *   Vue 3 前端脚手架搭建 (Vite, Pinia, Vue Router)
2.  **用户体系 (模块 1)**
    *   注册、登录 (JWT 认证)
    *   个人资料管理与身体档案录入
3.  **AI 教练初版 (模块 2)**
    *   集成 DeepSeek AI 接口
    *   实现基础的多轮问答功能
    *   基于用户身体档案生成初始训练计划和饮食方案
4.  **内容库基础**
    *   建立基础的动作库和食物库模型与数据结构

### 第二阶段：核心业务闭环与动态调整
**目标：** 实现打卡追踪，完善 AI 的动态调整能力，引入基础数据可视化。

1.  **训练与饮食管理 (模块 3 & 4)**
    *   每日训练打卡与饮食记录
    *   用户训练/饮食历史查询
2.  **AI 动态调整引擎 (模块 2)**
    *   根据用户的打卡反馈 (如未完成、疲劳度、体重变化)，AI 动态调整次日或后续的计划
3.  **数据中心初版 (模块 5)**
    *   体重、体脂趋势图
    *   简单的营养摄入统计

### 第三阶段：体验升级、留存与运营
**目标：** 完善附加功能，提高用户粘性，打造企业级体验。

1.  **社区与互动 (模块 7)**
    *   动态发布、信息流浏览
    *   点赞、评论、关注机制
    *   排行榜 (基于训练时长或积分)
2.  **成就与激励系统 (模块 6 & 1.5)**
    *   成就系统定义与检测逻辑
    *   勋章解锁与消息通知
3.  **高级功能探索**
    *   3D 肌肉视觉可视化 (结合训练计划)
    *   饮食视觉识别 (上传图片识别热量)
4.  **系统管理后台 (模块 8)**
    *   管理员对用户、内容的管理

## 三、 技术栈选型

*   **前端:** Vue 3, Vite, Pinia, Vue Router, Element Plus / Ant Design Vue (UI框架), ECharts (数据可视化)
*   **后端:** Java 17+, Spring Boot 3, Spring Security (认证授权), MyBatis-Plus (ORM)
*   **数据库:** MySQL 8.0 (持久化), Redis (缓存, 会话, 排行榜)
*   **AI 接口:** DeepSeek API (或 OpenAI 兼容接口)
*   **工具:** Maven, npm/yarn, Git

## 四、 风险评估与应对措施

1.  **AI 生成结果的不稳定性：** 需设计严谨的 Prompt，并引入后置的格式校验（如要求 AI 输出 JSON 格式，后端解析失败则重试或降级处理）。
2.  **功能过多导致延期：** 严格按照阶段划分，第一阶段未完成且稳定前，绝不开启第三阶段开发。
